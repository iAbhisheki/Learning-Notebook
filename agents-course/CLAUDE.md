# CLAUDE.md — AI Assistant Context for Deep Agents + RAG Course

## Project Overview

This is a **Deep Agents + RAG** learning course consisting of 24 Jupyter notebooks, 4 capstone projects, and supporting documentation. The course covers LLM fundamentals through production-grade agent systems.

## Directory Structure

```
agents-course/
├── 01-03: Phase 1 — LLM Fundamentals (API, structured output, embeddings)
├── 04-06: Phase 2 — RAG (chunking, hybrid search, vector DBs)
├── 07-08: Phase 3 — Tools & Function Calling (tool schemas, ReAct)
├── 09-11: Phase 4 — Agent Architecture (memory, planning, multi-agent)
├── 12-14: Phase 5 — Advanced Concepts (reflection, toolformer, filesystem)
├── 15-17: Phase 6 — Production Engineering (eval, guardrails, optimization)
├── 18-22: Phase 7 — Industry Frameworks (LangChain, LangGraph, LangSmith)
├── 23:    Phase 8 — Deep Agent Patterns (agentic RAG, MCP, A2A, swarm)
├── capstone/: 4 capstone project scaffolds
├── requirements.txt: All Python dependencies
├── .env.example: API key configuration template
├── SKILL.md: Study guide and learning paths
├── TOPICS_GUIDE.md: Complete concept reference
└── CLAUDE.md: This file
```

## Key Design Decisions

1. **Multi-model support**: All notebooks use `litellm` for model-agnostic code. Phase 7 notebooks also use native LangChain providers (`langchain-openai`, `langchain-google-genai`, `langchain-anthropic`).

2. **Interview readiness**: Every notebook includes `<details>` quiz blocks covering concepts frequently asked in AI/ML interviews.

3. **Programmatic generation**: Notebooks were generated by Python scripts (in `/tmp/gen_*.py`) that write `.ipynb` JSON directly. This ensures consistency and reproducibility.

4. **Progressive complexity**: Each phase builds on the previous one. Concepts from earlier notebooks are reused in later ones.

## How Notebooks Are Structured

Each notebook follows this pattern:
1. **Title + overview table** — What you'll learn
2. **Setup cell** — Imports, model configuration
3. **Theory sections** — Markdown with diagrams and comparison tables
4. **Code cells** — Runnable implementations
5. **Interview quiz** — `<details>` blocks with Q&A
6. **Build exercise** — Hands-on project
7. **Summary table** — Quick reference

## Important Notes

- **`.env` file required**: Copy `.env.example` to `.env` and add API keys
- **At least one LLM provider key** is needed (OpenAI recommended for widest compatibility)
- **LangSmith** (NB21) requires a separate API key from smith.langchain.com
- **Some frameworks** (CrewAI, AutoGen, LlamaIndex) are optional — notebooks gracefully handle missing installs
- **Capstone projects** are scaffolds with TODO stubs — learners implement them

## Common Tasks

### Adding a new notebook
1. Create a generator script in `/tmp/gen_XX.py`
2. Follow the standard cell structure (md + code + quiz + build)
3. Include multi-model setup in the first code cell
4. Add interview quiz with `<details>` blocks
5. Update README.md, TOPICS_GUIDE.md, and SKILL.md

### Modifying existing content
- Edit the generator script, re-run to regenerate the notebook
- Or edit the `.ipynb` directly (JSON format)

### Running the course
```bash
pip install -r requirements.txt
cp .env.example .env  # Add API keys
jupyter notebook
```
